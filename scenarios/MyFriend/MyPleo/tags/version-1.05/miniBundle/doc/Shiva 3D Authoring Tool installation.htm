The ShiVa Authoring Tool is a powerful software that allows to package your Shiva applications for various target platforms.
This is made possible because the code developed on the Shiva Editor is portable and can be executed on various target platforms without any changes.

The supported target platforms are:

  • Windows
  • Mac
  • Linux
  • iPhone
  • iPad
  • Android
  • Palm

According to the target platform, some additional third party tool need to be installed to allow building the applications for the given target platform. The objective of this document is to explain how to install the Authoring Tool and the required third party tools.

Please note that the screenshots have been taken from Windows but the documentation also applies to Mac OS and Linux.

System requirements for installation

Installing the ShiVa Authoring Tool under Windows

Windows XP or higher is required.

The ShiVa Authoring Tool for Windows is included in the ShiVa Editor Installater. To install it, check the option in ShiVa Editor Installer as follows :

You can also download the ShiVa Authoring Tool as a separate file at:
http://www.stonetrip.com/developer/download
Once installed, the Authoring Tool is available in the Startup Menu under Stonetrip > ShiVa Authoring Tool.
By default, the files are installed in Program Files\Stonetrip\ShiVa Authoring Tool.

Installing the ShiVa Authoring Tools under MacOS

Mac OS X 10.5 or higher is required.
Download the Authoring Tool for MacOS on the developer website at:
http://www.stonetrip.com/developer/download
Double click on the dmg to mount the volume then follow the installation steps.

Once installed, the Authoring Tool is available in the Applications Place under Stonetrip > Authoring Tool > ShiVa Authoring Tool.

Required third party tools

The ShiVa Authoring Tool has been designed to generate applications for various target platforms: Windows, MacOS, Linux, iPhone, Android, Palm, Wii, etc...

According to the target platform, some third party tools (not included in the installation for legals reasons ) are required to build the applications.

Required third party tools for Windows target

Visual Studio C++ Express

Note that Visual Studio C++ 2010 Express is free and only required to build a native Windows application.
Download the latest version at: http://www.microsoft.com/express/downloads/
Do a regular installation.

Microsoft .NET Framework

Download the latest version at: http://www.microsoft.com/downloads/
Do a regular installation.

Required third party tools for Mac target

Xcode

Download the latest version at: http://developer.apple.com/technologies/xcode
Just follow the installer steps, there is no special option.

Required third party tools for Linux target

No additional third party tool is required.

Required third party tools for iPhone/iPad target

Xcode

Download the latest version at: http://developer.apple.com/technologies/xcode
Just follow the installer steps, there is no special option.

iOS SDK

Download the latest version at: https://developer.apple.com/iphone/
Just follow the installer steps, there is no special option.

Required third party tools for Android target

Android SDK

Download the lastest version at: http://developer.android.com/sdk/index.html

Install as follows:

  • Unzip the file in a place like “C:\Android\SDK” on Windows (in that case, installer is “C:\Android\SDK\SDK Setup.exe”) or “Applications/Android/SDK” on OSX.
    Important : The path must not contain space.
  • Launch the SDK Setup :
    • If “Failed to fetch UR...” message is displayed at the first launch, go to the settings and check “Force https://... sources to be fetched using http://”
    • Click on Available Packages on the left and select the desired SDK Platform version (at least the SDK Platform Android 2.0), note that you will be able to select the minimum platform version on the Authoring Tool.
    • On Windows, you can select the USB Driver package to be able to push the application directly on connected Android devices.
    • Then click “Install Selected”

Android NDK

Download the latest version at: http://developer.android.com/sdk/ndk/index.html

Like for the SDK, unzip the file in a place like “C:\Android\NDK” on Windows ( in that case, installer is “C:\Android\NDK\ndk-build” ) or “Applications\Android\NDK” on OSX.

Important : The path must not contain space.

Cygwin ( Windows Only )

Download the latest version at: http://www.cygwin.com/

Install as follows:

  • Launch the installer.
  • Select “Install from Internet”.
  • Click next until a download site is asked, at this step, select any server.
  • When you are asked to select your package, expand the Devel category and select the binary make Package as follows:
  • Click Next and wait for the installation to complete.

Apache Ant ( Windows Only )

Download the latest version at: http://ant.apache.org/bindownload.cgi

Install as follows:

  • Unzip the zip file in a place like C:\Ant, to have C:\Ant\bin\ant.bat.
  • Add a new environment system variable ( Advanced system settings > Environment variables ) named ANT_HOME with the path where you unzipped Ant, as value.

Java SE Development Kit ( Windows Only )

Download the latest JDK version at: http://java.sun.com/javase/downloads/index.jsp
Install the JDK in the default directory (click Next until the end).

Configure the JAVA_HOME environment variable to the Java jdk path ( ex :C:\Program Files\Java\jdk1.6.0_20 ).

Eclipse

Eclipse environment is required to build android project

Download the latest version : http://www.eclipse.org/

Select Eclipse Classic, Eclipse 3.5 or newer is required.

ADT Plugin for Eclipse

Use Update Manager feature of your Eclipse installation to install the latest revision of ADT on your development computer.

  1. Start Eclipse, then select Help > Install New Software....
  2. Click Add, in the top-right corner.
  3. In the Add Repository dialog that appears, enter "ADT Plugin" for the Name and the following URL for the Location:
  4. https://dl-ssl.google.com/android/eclipse/
  5. Note: If you have trouble acquiring the plugin, try using "http" in the Location URL, instead of "https" (https is preferred for security reasons).
  6. Click OK.
  7. In the Available Software dialog, select the checkbox next to Developer Tools and click Next.
  8. In the next window, you'll see a list of the tools to be downloaded. Click Next.
  9. Read and accept the license agreements, then click Finish.
  10. When the installation completes, restart Eclipse.
Complete documentation for the ADT Plugin : http://developer.android.com/sdk/eclipse-adt.html

Required third party tools for Palm WebOS target

For more information about the installation please visit : http://developer.palm.com/

Cygwin ( Windows Only )

Note that Cygwin is only mandatory if you want to build a native Palm application.
To install, please refer to instructions above.

Palm SDK

Download the latest version at: http://developer.palm.com
Do a default installation (there is no special option).

Sourcery G++

Download the IA32 Windows Installer at: http://www.codesourcery.com/sgpp/lite/arm/portal/release858
Launch the installer and choose the Typical install set, then select the Modify PATH for all users.
